Example

Now follows a simple example which excersizes some of the features of Aapp.

Layout

	  build.xml
	+-src
	  +-com
	    +-schau
	      +-AappTest
	          AappTest.java
                  include.j		   
                  README.txt
              +-EmptyDirOnPurpose
	      +-JunkDir
	          README.txt

The example will show that files/directories in the EmptyDirOnPurpose and JunkDir are not touched.

The actual source code is in the src/com/schau/AappTest/AappTest.java file. The include.j and README.txt files are files which gets included into AappTest.java during processing.

The build.xml are the main Ant project file and is shown below.

build.xml

        <?xml version="1.0" encoding="ISO-8859-1"?>
        <project name="Test" basedir="." default="all">
            <!-- HINT: read this file from the bottom and up! -->

            <!-- Clean up targets -->
            <target name="clean" description="Delete all generated files">
                <delete verbose="true" failonerror="false" includeEmptyDirs="true">
                    <fileset dir="." defaultexcludes="no">
                        <include name="**/*~" />
                    </fileset>
                </delete>
                <delete dir="final" />
                <delete dir="classes" />
            </target>

            <!-- preprocess the files -->
            <target name="preprocess" description="Use">
                <!-- taskdef the Aapp task. I have my Ant tasks stored in
                        /opt/AntTasks. The taskdef'ing can be moved out into
                        its own target and called once - if you have multiple
                        aapp task. -->
                <taskdef name="aapp" classname="com.schau.Aapp" classpath="/opt/AntTasks/Aapp.jar" />
                <aapp srcdir="src" dstdir="final" showversion="true">
                    <!-- declaring som variables - they're not used -->
                    <variable name="var1">var1Value</variable>
                    <variable name="var2">var2Value</variable>
                    <variable name="var3">var3Value</variable>
                    <variable name="var4">var4Value</variable>
                </aapp>
            </target>

            <!-- compile the classes stored in final -->
            <target name="compile" depends="preprocess">
                <mkdir dir="classes" />
                <javac srcdir="final/com/schau/AappTest" destdir="classes">
                    <compilerarg value="-Xlint:unchecked" />
                </javac>
            </target>

            <!-- run the compiled class -->
            <target name="run">
                <java classname="com.schau.AappTest.AappTest" classpath="classes" />
            </target>

            <!-- default target: preprocess the files, compile the files
                 and then run the compiled binary -->
            <target name="all" depends="compile,run" />
        </project>

AappTest.java

        #include "include.j"

        /**
         * Main entry class.
         */
        public class AappTest {
            /**
             * main
             */
            public static void main(String[] args) {
                System.out.println("#=GREETING from #=VERSTR");
                System.out.println("LICENSE: #include 'README.txt'");
            }
        }

A quick glance at the above source code reveals that there is two variable insertions (#=GREETING and #=VERSTR) and two file inclusions (#include "include.j" and #include 'README.txt') to be done.

include.j

        package com.schau.AappTest;

        /**
         * This comment block will be carried over to the main program.
         */
        #define VERSION 1
        #rem However, this comment block is secret
        #define NAME "AappTest"
        #define VERSTR NAME##" v"##VERSION
        #rem ... and so is this :-)
        #define GREETING "Hello, world"
        #rem the end.

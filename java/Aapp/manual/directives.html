Directives

This chapter covers all arithmetic operators, relational operators and other directives that Aapp understands.

General Rules

  • True is 1, false is 0.
  • If you try to add two strings they will be concatenated: "A" + "B" = "AB".
  • If you try to add a string and an integer the integer will be converted to a string and the two strings will be concatenated: 1 + "A" = "1A".
  • Adding two integers will have the expected outcome of one integer: 1 + 1 = 2.
  • Doing other kinds of maths on strings will make Aapp exit with an error. Thus, this is illegal: 1 / "A".
  • Equality tests (==, !=, <, <=, >=, >) can be done on strings and integers of the same type. Comparing an integer to a string will fail.
  • Strings must be embedded in either single quotes (') or double quotes (").

Arithmetic Operators

op1 + op2 Add the two operands - see General Rules above for special cases.
op1 - op2 Subtract op2 from op1. Both must be integers.
op1 * op2 Multiply op1 with op2. Both must be integers.
op1 / op2 Divide op1 by op2. Both must be integers. An error is thrown if op2 is 0.
- op Unary minus - negate op. Op must be an integer. Note there is currently no + op.
( ... ) Standard paranthesis.
op1 & op2 Calculates op1 AND op2.
op1 | op2 Calculates op1 OR op2.
op1 ^ op2 Calculates op1 XOR op2.

Relational Operators

! op Not op. If op is non-zero, the result is 0. If op is zero the result is 1.
op1 (==, !=, <, <=, >=, >) op2 Equality test. Returns 1 the expression is true, 0 otherwise.
op1 && op2 Returns 1 if both op1 and op2 are true, 0 otherwise.
op1 || op2 Returns 1 if either op1 or op2 are true, 0 otherwise.

String Concatenation

There are two ways two concatenate strings:

  1. By adding (+) the strings.
  2. By using the concatenation operator: (##): "aa" ## "bb" = "aabb".

Variables Manipulation

define VARIABLE expression Set VARIABLE to the value of expression. Expression can be a simple expression such as a string or integer or a compound expression consisting of a mix of strings, integers and any of the arithmetic or relational operators.
redef VARIABLE expression Redefine a previous defined variable.
undef VARIABLE Undefine a previous defined variable.
=VARIABLE Insert the value of VARIABLE here.

Conditionals

if expression Execute the following block if the expression is true.
ifdef VARIABLE Execute the following block if the VARIABLE is defined.
ifndef VARIABLE Execute the following block if the VARIABLE is not defined.
elif expression If the previous if evaluted to false evaluate this expression and execute the following block if true.
else If the previous if, elif, ifdef or ifndef evaluted to false execute this block.
endif Terminate the previous if, ifdef or ifndef block.

File Operations

include FILE Include the content of FILE here. FILE can be a string ("include.j") or a variable:
A = "include.j"
include A
... will include the file "include.j". If a full path is not given the file will be searched for in the current directory and then in the directory list given by the includes attribute to Aapp.
run EXPRESSION Evaluate EXPRESSION and then run an external command. If EXPRESSION evaluates to "ls -l /root" the ls command will be run with the arguments -l /root. The return code from the command is stored in the $$ variable. Any output generated by the external command can be read from the $_ variable.

Miscellanous

echo EXPRESSION Evaluate EXPRESSION and write the result to stdout.
end Terminate parsing of this file.
rem COMMENT Add a comment to the include file. The comment will not be carried over to the processed file. The comment spans to the rest of the line.

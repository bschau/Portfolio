Usage

TaskDef

Before using Aapp you need to define it as a task within your project:

        <target name="init" description="Taskdef the Aapp task">
            <taskdef name="aapp" classname="com.schau.Aapp" classpath="/path/to/Aapp.jar" />
	</target>

The target attributes are not that important but pay attention to the taskdef attributes! name (aapp) is the name you want to use when referring to the Aapp task. classname must be com.schau.Aapp. If you copied the Aapp.jar to Ants lib directory you can omit the classpath attribute - if not, you have to enter the full path to the Aapp.jar.

Remember to taskdef before use!

Synopsis

	<aapp showversion="true"
	      srcdir="src"
	      dstdir="final"
	      includes="."
	      ppid="#"
	      suffixes=".java" >
	    <variable name="var1name">var1value</variable>
	    <variable name="var2name">var2value</variable>
                                               .
                                               .
                                               .
	    <variable name="varXname">varXvalue</variable>
        </aapp>

Aapp accepts the following attributes:

showversion Show version information when running Aapp. To show version information set to true or on or 1 or yes. Other values hides version information. Default is true.
srcdir Where to look for sources. Only one directory can be given - if more is needed, run Aapp multiple times. Default is src.
dstdir Where to store the processed files. The directory hierarchy of the scrdir will be preserved. To compile the files launch javac and point it to dstdir. Default is final.
includes Where to find include files. Multiple include paths can be given (separated by File.pathSeparator - look it up for your platform). Default is ..
ppid Pre-processor token. This is the anchor point of all the directives. Setting ppid to @@ means that #include should be spelled @@include instead. Default is #.
suffixes Suffix of files to process. Multiple suffixes can be given separated by |. Default is .java.

Variables

To define variables to be pre-loaded into Aapp before processing files embed one or more <variable> ... </variable> statements within the <aapp> ... </aapp> block:

        <variable name="varname">varvalue</variable>

... this will set the variable varname with the value varvalue

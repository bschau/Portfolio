Variable Scope

There are four variable scopes in Aapp. Three of these are read-only and the last is read/write.

The four scopes are (from innermost to outermost):

local: Inner scope. Variables in the local: scope can be set, read and deleted.
ant: Properties defined in the Ant build files. These are read only.
rt: The Java runtime variables (such as os.name and os.user). These are read only.
env: Environment variables (such as $PATH and $LOGNAME). These are read only.

If you try to redefine a ant:, rt: or env: variable the variable will be set in the local: scope.

F.ex. defining f.ex. os.name will create a copy of os.name in the local: scope so you will be able to:

        #=local:os.name
        #=rt:os.name

... and get two different answers!

If a scopeless variable is requested it will be looked for in the following scope order:

  1. local:, if not found then in
  2. ant:, if not found then in
  3. rt:, if not found then in
  4. env:, if not found then an error is generated.
